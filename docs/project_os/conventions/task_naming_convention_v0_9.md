# Task Naming Convention v0.9

## Format

`<Project Name> – <Phase> – <Role>`

## Examples

- `Example Project – Phase1.0 – Implementation`
- `Example Project – Phase1.0 – Project Core Review`

## Rules

- Project Nameを固定
- Phaseを含める
- RoleをCanonical Termで表す
- 表示名はIdentityの補助であり、AuthorityはRuntime / Registryで確認する
- Work UIで自動命名された場合、変更可能なら手動で正規化する

## Project Customization

使用環境にタスク名変更機能があるか記録してください。
