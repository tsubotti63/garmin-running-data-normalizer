# Governance Lifecycle Guide

## AI Collaboration Platform

**Version:** v0.2  
**Status:** Canonical Guide

---

# 1. Purpose

This guide defines the canonical lifecycle for projects adopting the Project Governance Extension.

All governance documents shall reference this guide instead of redefining lifecycle stages.

---

# 2. Lifecycle Overview

```text
Project Factory
      │
      ▼
Governance Onboarding
      ▼
Active Operation
      ▼
Review & Improvement
      ▼
Platform Improvement Intake
      ▼
Platform Evolution
      ▼
Archive / Retirement
```

---

# 3. Lifecycle Stages

## 3.1 Governance Onboarding

Objective:
- Establish Governance Profile
- Confirm operational roles
- Register ADRs where appropriate

Exit Criteria:
- Governance Profile approved
- Initial operational modules selected

---

## 3.2 Active Operation

Typical activities:

- Repository Operations
- Documentation Operations
- Release Operations
- Community Operations
- Maintenance Operations

Outputs:
- Operational evidence
- Review records
- Improvement candidates

---

## 3.3 Review & Improvement

Purpose:
- Evaluate operational effectiveness
- Identify governance improvements
- Produce recommendations

Outputs:
- Review reports
- Improvement backlog
- Candidate ADRs

---

## 3.4 Platform Improvement Intake

Reusable improvements are submitted to ACP.

Project-specific improvements remain within the project unless promoted.

---

## 3.5 Platform Evolution

Platform Evolution evaluates improvement candidates and determines whether ACP standards should be updated.

---

## 3.6 Archive / Retirement

Required outputs:

- Final governance summary
- Outstanding issues
- Lessons learned
- Reference preservation

---

# 4. Roles

| Role | Responsibility |
|------|----------------|
| Human Owner | Final governance decisions |
| AI Assistant | Analysis, drafting, review support |
| Reviewer | Independent governance validation |

---

# 5. Lifecycle Principles

- Human accountability remains explicit.
- Reviews are evidence-based.
- Improvements should be reusable where possible.
- Architectural boundaries must be preserved.
