# Public Provenance and Authority Reference

## Current public operation

- The repository is public on GitHub with default branch `main`.
- The project is licensed under `Apache-2.0`; the root `LICENSE` is canonical.
- GitHub Actions is operational and public maintenance is active.
- No Git tag, GitHub Release, or versioned release exists at this handoff.

## Historical pre-public context

- The Target repository was initialized independently with fresh Git history.
- A private predecessor was inspected read-only; its exact name, repository
  path, task identifiers, workflow history, data, and Git history are excluded.
- Local responsibility-level reuse was Human-authorized after dependency and
  privacy review.
- Target code uses the Target namespace and has no predecessor runtime/import
  dependency.
- At the initial pre-public handoff, the Target license, GitHub publication, and
  broader redistribution-rights decisions were unresolved. Apache-2.0 adoption
  and repository publication are now complete; they do not retroactively
  establish rights for predecessor or third-party material whose rights remain
  unconfirmed.

## Current handoff authority

- The Target Project Core Review role is the only active project gate. Prior
  extraction reviews are historical context, not active approval dependencies.
- Each maintenance push requires its applicable Human authorization together
  with the normal review and CI process; public status does not create standing
  push permission.
- Tags, GitHub Releases, versioned releases, and other Human-reserved actions
  require their applicable authorization and current release evidence.
